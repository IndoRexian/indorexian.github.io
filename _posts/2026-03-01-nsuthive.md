## The Problem
Since I first joined my university, whenever I wanted to get to know about a certain professor, I would go onto the [NSUT Sub](https://www.reddit.com/r/NSUT_Delhi/) and would search their name to find out more about them. The thing which irritated me a lot was I had to search a lot sometimes to find certain posts.

This actually gave me an idea to make a platform which did exactly that but in a more centralized way. And that's how [NSUT Hive](https://nsuthive.com) came into being.

## Starting Out
The first task was to actually scrape the data from a reliable source for all professors and after some research I found out that the [official website of NSUT](https://nsut.ac.in) contained this same information publically. However, I don't think so that this data gets updated regularly because many professors data was not available. I was able to get whatever data I could scrape very easily using [Selenium](https://selenium-python.readthedocs.io/). 

I have plans to figure out a way to get data from a more frequently updated source and to also have a page where users could report any missing professor data and I would be able to add it manually. 

While the initial plan was to use React, I later switched to NextJS as I wanted to learn something robust, and SSR was something that intrigued me.
